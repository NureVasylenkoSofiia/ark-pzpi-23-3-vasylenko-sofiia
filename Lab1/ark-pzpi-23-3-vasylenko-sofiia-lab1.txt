3 ОПИС ВИКОНАНОЇ РОБОТИ
3.1 Вступ
Під час виконання першої лабораторної роботи було створено документ Vision and Scope для системи автоматизації мережі кав’ярень. Документ допоміг визначити цілі, ключові функції та межі системи, а також окреслити архітектуру, технології та етапи розробки. Це дозволяє уникнути невизначеності на наступних етапах та чітко планувати реалізацію серверної частини, IoT-клієнта, веб-інтерфейсу та мобільного додатку.
3.2 Background
Власники кав’ярень та менеджери часто стикаються з проблемою ручного обліку замовлень, контролю залишків на складі та управління персоналом. Необхідна система, яка автоматично об’єднує дані з різних точок продажу та smart-пристроїв, спрощує управління мережею та допомагає приймати рішення на основі актуальної інформації.
3.3 Business Opportunities
Існуючі рішення (Poster, Square, Toast) частково автоматизують облік продажів та роботу з POS-терміналами, але не забезпечують повної інтеграції з IoT-пристроями, централізованого управління мережею та гнучкої аналітики. Наша система вирізняється об’єднанням усіх ключових процесів в єдине середовище та наявністю мобільного клієнта для оперативного доступу до даних.
3.4 Vision Statement
Система дозволяє централізовано управляти мережею кав’ярень, обробляти замовлення, контролювати залишки на складі, керувати персоналом та інтегрувати smart-пристрої, надаючи веб- та мобільний інтерфейс для швидкого доступу до актуальних даних.
3.5 Major Features
Система об’єднує управління закладами, облік замовлень і складських запасів, управління користувачами, інтеграцію з IoT-пристроями, формування базової аналітики, адміністрування системи та підтримку багатомовного інтерфейсу. Мобільний додаток забезпечує доступ до ключових показників та стану обладнання в реальному часі.
3.6 Scope of Initial Release
Сервер приймає та обробляє дані замовлень та залишків на складі, зберігає їх у базі даних і формує прості звіти. IoT-клієнт передає стан кавомашин та інших smart-пристроїв. Веб-інтерфейс дозволяє переглядати замовлення, склад і звіти, а мобільний додаток відображає основні показники та стан обладнання.
3.7 Scope of Subsequent Releases
У наступних версіях передбачено розширену аналітику, детальні звіти по продажах, повідомлення про критичні залишки, інтеграцію з платіжними сервісами, розширену підтримку IoT-пристроїв та додаткові мови інтерфейсу.
3.8 Operating Environment
Сервер реалізовано на ASP.NET Core з C#, база даних — PostgreSQL через Entity Framework Core. IoT-клієнт працює на C# і взаємодіє з сервером через HTTP API та WebSocket. Веб-фронтенд розроблено на React.js, асинхронні запити здійснюються через Fetch API, а мобільний додаток надає оперативний доступ до ключових даних.
ВИСНОВКИ

У ньому визначено цілі та ключові функції системи, включно з серверною частиною, IoT-клієнтом, веб та мобільним інтерфейсом. Було окреслено межі первинного випуску та потенційні можливості для наступних версій, а також проаналізовано існуючі рішення на ринку та виділено переваги нашого продукту. Визначено робоче середовище системи та технології, які будуть використовуватися для реалізації прототипу. У результаті роботи сформовано чітке уявлення про структуру та функціональність системи, що дозволяє планувати подальші етапи розробки та уникнути невизначеності на наступних лабораторних роботах.

1. Business Requirements (Бізнес-Вимоги)
   1. Background (Передумови)
Сучасні мережі кав’ярень функціонують в умовах високої конкуренції, зростаючих очікувань клієнтів та необхідності оперативного управління бізнес-процесами. Значна частина закладів використовує розрізнені програмні рішення для обробки замовлень, обліку складу, управління персоналом та аналізу продажів, що призводить до ускладнення управління, втрати даних та зниження ефективності роботи.
У зв’язку з цим виникає потреба у створенні комплексної програмної системи, яка забезпечить автоматизацію ключових бізнес-процесів мережі кав’ярень, підвищить ефективність управління, забезпечить можливість масштабування та створить основу для подальшої комерціалізації продукту.
   2. Business Opportunity (Бізнес-можливості)
Ринок закладів громадського харчування, зокрема мереж кав’ярень, активно зростає та цифровізується. Власники бізнесу все більше зацікавлені в автоматизації операційних процесів, підвищенні прозорості управління та використанні аналітики для прийняття рішень. Це створює попит на сучасні програмні системи, здатні об’єднати всі ключові процеси в єдине інформаційне середовище.
На ринку вже існують рішення для автоматизації кав’ярень, які переважно зосереджені на обліку продажів та роботі з POS-терміналами. Проте більшість таких систем мають обмежені можливості інтеграції з IoT-пристроями, недостатню гнучкість при масштабуванні або потребують використання декількох окремих сервісів для повноцінного управління мережею закладів.
Цільовою аудиторією програмної системи є власники та менеджери малих і середніх мереж кав’ярень, які потребують централізованого інструменту для управління кількома закладами, а також персонал кав’ярень, що безпосередньо працює з замовленнями, складом та обладнанням. Система орієнтована на масового користувача та не потребує глибоких технічних знань для повсякденної роботи. 
Серед існуючих конкурентних рішень можна виділити такі системи:
1. Poster ( https://joinposter.com/ua) – забезпечує базову автоматизацію продажів та складського обліку, але має обмежені можливості гнучкого масштабування та розширеної інтеграції з IoT-пристроями.
2. Square (https://app.squareup.com) – орієнтований переважно на платіжні операції та POS-функціональність, що робить його менш зручним для комплексного управління мережею закладів.
3. Toast (https://pos.toasttab.com) – пропонує розширені функції для ресторанного бізнесу, проте є складним у налаштуванні та менш адаптованим до потреб невеликих мереж кав’ярень.
Запропонована програмна система вирішує проблеми фрагментованості даних, відсутності централізованого контролю, складності масштабування та обмеженої аналітики. Вона поєднує управління бізнес-процесами, інтеграцію зі smart-пристроями та аналітичні інструменти в межах єдиного програмного рішення.
2. Vision of the Solution (Концепція Рішення)
   1. Vision Statement (Окреслення Концепції)
Програмна система призначена для комплексної автоматизації управління мережею кав’ярень шляхом об’єднання ключових бізнес-процесів у єдине цифрове середовище. Рішення покликане спростити операційну діяльність, підвищити прозорість управління та забезпечити швидке прийняття управлінських рішень на основі актуальних даних.
Система забезпечує централізований контроль замовлень, складських запасів, персоналу та обладнання, а також підтримує інтеграцію з IoT та smart-пристроями. Це дозволяє власникам і менеджерам ефективно керувати мережею закладів незалежно від їх кількості та географічного розташування.
Використання програмної системи створює середовище, у якому бізнес-процеси кав’ярень стають більш автоматизованими, масштабованими та адаптивними до змін ринку, водночас зберігаючи простоту використання для персоналу та відповідність сучасним вимогам безпеки й міжнародним стандартам.
   2. Major Features (Головна функціональність)
1. Управління закладами мережі. Створення та управління кількома кав’ярнями в межах однієї системи.
2. Обробка замовлень. Створення, перегляд та оновлення замовлень, збереження історії продажів.
3. Складський облік. Облік продуктів та інгредієнтів, контроль залишків після кожного продажу.
4. Управління користувачами. Реєстрація користувачів, розподіл ролей (адміністратор, менеджер, персонал).
5. Інтеграція з IoT / smart-пристроями. Отримання базових даних зі smart-пристроїв (наприклад, стан кавомашини).
6. Аналітика. Перегляд простих звітів про продажі та популярні позиції меню.
7. Адміністрування системи. Управління даними системи.
8. Багатомовний інтерфейс. Підтримка української та англійської мов.

4. Business Context (Бізнес-Контент)
4.3 Operating Environment (Робоче середовище)
Система працює на серверній платформі .NET з використанням Web API для обміну даними між сервером та клієнтськими модулями. Основна бізнес-логіка обробки замовлень, управління складом і користувачами реалізована на C#. Сервер взаємодіє зі смарт-пристроями кав’ярень (каво-машини, POS-термінали) через HTTP API та WebSocket для отримання актуальних даних у реальному часі.
Для зберігання даних використовується PostgreSQL, де ведеться облік замовлень, залишків на складі, інформації про персонал та налаштувань закладів. ORM шар реалізований за допомогою Entity Framework Core, структура бази даних організована у вигляді окремих таблиць для кожної логічної сутності: замовлення, товари, користувачі, заклади та пристрої.
Локальні IoT-клієнти розробляються на C# і забезпечують передачу базових даних про стан кавомашин та актуальні замовлення. Дані передаються у форматі JSON через HTTP API або WebSocket для своєчасного оновлення стану системи.
Веб-інтерфейс фронтенду побудований на React та адаптований для роботи на десктопних браузерах і планшетах, які можуть використовуватися на точках продажу. Для навігації між екранами застосовується React Router, а ключові дані відображаються у компактному та зручному вигляді для швидкого перегляду менеджерами та персоналом. UI оптимізовано для невеликих екранів POS-терміналів і планшетів за допомогою CSS Modules, що забезпечує чіткі шрифти та контрастні індикатори.
Система спроектована з урахуванням високої продуктивності та надійності. API обробляє запити з мінімальною затримкою, а сервер виконує агрегацію даних у фонових процесах. Основна аналітика оптимізована для інкрементальних обчислень, що дозволяє швидко отримувати актуальні звіти про продажі, залишки на складі та ефективність роботи закладів.
